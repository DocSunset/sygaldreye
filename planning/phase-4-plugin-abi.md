# Phase 4: Plugin ABI

## Goal

Every node type — including built-ins — satisfies a stable C-linkage ABI that enables
`dlopen()` loading, LLM-authored hot-loading over WiFi, and cross-compilation without
restarting the app. This is the architectural foundation everything else builds on;
it must be designed before the runtime graph (Phase 5).

## The ABI

```c
// eyeballs_node_abi.h  (C, not C++)
typedef struct {
    int   version;          // ABI version; reject if != EYEBALLS_ABI_VERSION
    const char* type_name;  // e.g. "water_surface"
    const char* description;

    void*       (*create)   ();
    void        (*destroy)  (void* node);
    void        (*process)  (void* node, double time_s);
    const char* (*serialize)(void* node);          // → malloc'd JSON string
    void        (*free_str) (const char* s);       // companion to serialize
    void        (*deserialize)(void* node, const char* json);
} EyeballsNodeDescriptor;

#define EYEBALLS_ABI_VERSION 1

// Every plugin .so exports this symbol:
extern "C" const EyeballsNodeDescriptor* eyeballs_node_descriptor();
```

`serialize`/`deserialize` are auto-generated by the PFR+sygaldry machinery from Phase 1.
`process` maps to `operator()(double time)`.

## Code generation

A single C++ template wraps any sygaldry-style node into the descriptor:

```cpp
template<typename Node>
const EyeballsNodeDescriptor* make_descriptor() {
    static EyeballsNodeDescriptor d {
        .version     = EYEBALLS_ABI_VERSION,
        .type_name   = Node::name(),
        .create      = []() -> void* { return new Node{}; },
        .destroy     = [](void* p) { delete static_cast<Node*>(p); },
        .process     = [](void* p, double t) { (*static_cast<Node*>(p))(t); },
        .serialize   = [](void* p) -> const char* {
                           auto s = to_json(*static_cast<Node*>(p));
                           char* out = strdup(s.c_str());
                           return out;
                       },
        .free_str    = [](const char* s) { free(const_cast<char*>(s)); },
        .deserialize = [](void* p, const char* json) {
                           from_json(*static_cast<Node*>(p), json);
                       },
    };
    return &d;
}

// In the plugin .cpp:
extern "C" const EyeballsNodeDescriptor* eyeballs_node_descriptor() {
    return make_descriptor<WaterSurface>();
}
```

## Dynamic loading

```cpp
// component_registry.cpp
void ComponentRegistry::load_plugin(const std::string& path) {
    void* handle = dlopen(path.c_str(), RTLD_NOW | RTLD_LOCAL);
    auto* fn = reinterpret_cast<decltype(&eyeballs_node_descriptor)>(
                   dlsym(handle, "eyeballs_node_descriptor"));
    const auto* desc = fn();
    if (desc->version != EYEBALLS_ABI_VERSION) { dlclose(handle); return; }
    registry_[desc->type_name] = {desc, handle};
}
```

## Cross-compile pipeline (companion)

```python
# companion: compile a node from source
def compile_node(src_path: str) -> bytes:
    ndk = os.environ["ANDROID_NDK_ROOT"]
    cc  = f"{ndk}/toolchains/llvm/prebuilt/linux-x86_64/bin/aarch64-linux-android33-clang++"
    out = src_path.replace(".cpp", ".so")
    subprocess.run([cc, "-shared", "-fPIC", "-std=c++20",
                    "-I", "sygaldry/sygaldry/sygah-endpoints",
                    src_path, "-o", out], check=True)
    return open(out, "rb").read()
```

The LLM writes a `.cpp` file that includes `eyeballs_node_abi.h` and the sygaldry headers.
The companion compiles and uploads it. The headset loads it. No restart.

## Key Design Decisions

**C linkage boundary is non-negotiable.** C++ has no stable ABI — vtables, name mangling,
and exception handling differ between compiler versions. A C struct of function pointers
is the only portable choice, as proven by VST2, PD externals, and VCV Rack.

**ABI versioning from day one.** The `version` field allows the runtime to reject
incompatible plugins gracefully rather than crashing. Bump it whenever the descriptor
struct changes.

**Built-ins satisfy the same ABI.** There is no "native fast path" for built-in node
types. Everything goes through the descriptor. This simplifies the runtime graph
(one code path) and ensures built-ins are always portable.

**`RTLD_LOCAL` not `RTLD_GLOBAL`.** Plugins should not pollute each other's symbol
namespaces. Each plugin gets its own isolated handle.
