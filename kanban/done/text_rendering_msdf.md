# MSDF text rendering: TextMesh component

Implement a `TextMesh` component in the `render` package — the text analogue of `CubeMesh` — that renders sharp, resolution-independent text using Multi-channel Signed Distance Fields (MSDF). Text must remain crisp at any viewing distance, including a few millimetres away in VR.

## Dependencies

- `msdf-atlas-gen` via nixpkgs — offline atlas generation
- `nlohmann_json` via nixpkgs — parse atlas metrics JSON at startup

## Approach

1. **Offline atlas generation**: use `msdf-atlas-gen` (via Nix) to produce a PNG glyph atlas and JSON metrics from a chosen TTF. Run once per font; commit the outputs as assets.
2. **`TextMesh` component** (in `render` package): load the atlas as a GL texture, parse glyph metrics (UV rect, advance, bearing). Owns the MSDF shader pair and glyph layout. Exposes a `draw(string, world_transform)` call — no GL knowledge required by callers.
3. **Glyph layout** (owned by `TextMesh`): pure data transform — string + metrics → list of `(quad position, UV rect)` pairs. No GL dependency; could be extracted to a helper if it grows large.
4. **MSDF shader pair**: vertex shader for textured quads + fragment shader (sample 3 channels, compute median, `smoothstep` → alpha). ~10 lines for the fragment shader.

## Lighting policy

Text is **emissive (unlit)**. The MSDF fragment shader outputs the text color directly with no light attenuation. Rationale: text legibility must be consistent regardless of viewing angle or scene lighting — a label that's dark because it's facing away from the light is a bug, not a feature. This is an intentional design decision, not an oversight; the design doc should record it.

## Acceptance

- `TextMesh::draw(string, transform)` renders the string sharp at 5 cm and at 2 m simulated viewing distance
- Text brightness is uniform regardless of the label's orientation relative to scene lights
- The atlas and metrics are generated by a reproducible offline step documented in the component or build scripts
- `TextMesh` follows project conventions: `render` package, under 100 lines of substantive code per file, with a design doc
- No runtime font rasterization; all rasterization is offline
- Callers need no GL knowledge to draw text
