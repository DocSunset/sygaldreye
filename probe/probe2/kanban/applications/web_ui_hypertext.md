# Web UI components + web content in the graph (brainstorm 2026-06-20)

Direction note, not yet a work item.

The graph should eventually integrate web UI components and web content,
so the library can author pedagogical interactive hypertexts — graph
nodes as live, embedded web widgets / hypertext fragments, wirable into
the rest of the patch like any other node.

Open questions: rendering surface (in-VR browser/CEF vs the existing
browser build target), how DOM interaction maps to ports/edges, and
whether "web content" means embedding pages or authoring them from the
graph.
